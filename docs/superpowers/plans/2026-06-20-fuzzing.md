# Fuzzing harness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Zig-native fuzz tests for `parseHead` and `decodeInPlace` that assert arbitrary input never panics, with a corpus smoke under plain `zig build test` and real fuzzing via `zig build test --fuzz`.

**Architecture:** Two `std.testing.fuzz` tests in the parsers' own test blocks; each draws fuzz bytes via `smith.sliceWithHash`, feeds them to the target, and treats any error/result as a pass (only a panic/UB fails). Seed corpora make the in-suite smoke meaningful.

**Tech Stack:** Zig 0.16 native fuzzing (`std.testing.fuzz` / `Smith`).

## Global Constraints

- Zig 0.16. Additive test code only — no `src/` library/behavior change.
- `std.testing.fuzz({}, testOne, .{ .corpus = &.{...} })`; `testOne: fn(void, smith: *std.testing.Smith) anyerror!void`.
- `Smith` has only `*WithHash` draws → fill a scratch buffer via `smith.sliceWithHash(&buf, <distinct u32>) u32`; one distinct hash constant per call site.
- Contract = **no panic / no UB**: any `ParseError` / `DecodeResult` is a pass. Use `_ = target(...) catch {};` for the fallible one; `_ = decodeInPlace(...)` for the non-fallible one.
- Tests live in the module test blocks (`src/http/parser.zig`, `src/http/chunked.zig`) so `zig build test` runs a corpus smoke (CI covers it); `zig build test --fuzz` runs the fuzzer.
- The seed corpus is curated-safe → the smoke is expected green; a corpus crash = real bug (separate fix).
- Test baseline: **current main passes** (`zig build test --summary all`, 0 failures); expect +2 tests. `timeout` is GNU-only / not on this mac — run zig directly.

- [ ] **Step 1: Add the parseHead fuzz test**

In `src/http/parser.zig`, in the test block (alongside existing parser tests), add. `std`,
`request`, `Header`, and `parseHead` are already in scope in this file:

```zig
test "fuzz: parseHead never panics on arbitrary input" {
    try std.testing.fuzz({}, struct {
        fn one(_: void, smith: *std.testing.Smith) anyerror!void {
            var buf: [8192]u8 = undefined;
            const n = smith.sliceWithHash(&buf, 0x9001);
            var hs: [request.max_headers]Header = undefined;
            _ = parseHead(buf[0..n], &hs) catch {}; // any ParseError ok; contract = no panic
        }
    }.one, .{ .corpus = &.{
        "GET / HTTP/1.1\r\nHost: x\r\n\r\n",
        "POST /e HTTP/1.1\r\nContent-Length: 5\r\n\r\nhello",
        "POST /e HTTP/1.1\r\nTransfer-Encoding: chunked\r\n\r\n",
        "BADLINE",
        "GET / HTTP/1.1\r\n" ++ ("X: y\r\n" ** 80) ++ "\r\n",
        "",
    } });
}
```

- [ ] **Step 2: Add the decodeInPlace fuzz test**

In `src/http/chunked.zig`, in the test block, add. `std` and `decodeInPlace`/`DecodeResult`
are in scope. `buf` must be mutable (decode is in place):

```zig
test "fuzz: decodeInPlace never panics on arbitrary input" {
    try std.testing.fuzz({}, struct {
        fn one(_: void, smith: *std.testing.Smith) anyerror!void {
            var buf: [8192]u8 = undefined;
            const n = smith.sliceWithHash(&buf, 0x9002);
            _ = decodeInPlace(buf[0..n], 4096); // bounded max
            const n2 = smith.sliceWithHash(&buf, 0x9003);
            _ = decodeInPlace(buf[0..n2], 0);   // unbounded max
        }
    }.one, .{ .corpus = &.{
        "5\r\nhello\r\n0\r\n\r\n",
        "5;ext=1\r\nhello\r\n0\r\nX-T: 1\r\n\r\n",
        "0\r\n\r\n",
        "zz\r\nbad\r\n",
        "fffffffffffffffe\r\nhi\r\n0\r\n\r\n",
        "",
    } });
}
```
